**四、 遇到的问题与困难**

*   **技术问题：**
    *   **数据库迁移幂等性**：SQLite 的 `ALTER TABLE ADD COLUMN` 没有 `IF NOT EXISTS` 语法，旧版本 schema 已有该列时直接崩溃。解决方案：封装 `_safeAddColumn()` 用 try-catch 静默跳过。
    *   **鸿蒙端 `flutter_secure_storage` 不识别 ohos 平台**：原生插件已注册但 Dart facade 不分发到 ohos。解决方案：安全存储调用点改用 `flutter_secure_storage_ohos` facade。
    *   **Android 构建 JDK 冲突**：Gradle 使用了 DevEco Studio 的 JBR 而非系统 JDK17。根因是 DevEco 启动的旧 Gradle daemon 缓存了错误 JDK 路径，`./gradlew --stop` 清理后恢复正常。
    *   **协作同步根因排查**：全链路涉及 Flutter Dart / Go Socket.IO / PostgreSQL / MinIO 四层，需逐层审计。通过三份独立 Agent 调研交叉验证，去重后确认 7 个根因。
    *   **已尝试的解决方案：** 查阅 SQLite 官方文档、Socket.IO v4 投递保证文档、Figma 多人协作技术博客、Excalidraw CRDT 迁移 RFC、Yjs 文档
    *   **需要帮助或待解决：** 协作 Phase 1 需要前后端联调（Go Socket.IO ACK/CSR POC），届时需要服务端配合

*   **非技术问题：**
    *   鸿蒙 `flutter_ohos` 是社区移植版，部分 API 与标准 Flutter 不一致（FragmentProgram/Shader 不支持、路径约定不同），需要逐个验证，效率低于标准平台。
    *   **已采取的措施：** 建立 `harmonyos-guides/` 知识库记录适配经验，Agent 文档中标注鸿蒙特殊处理点
