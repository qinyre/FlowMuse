# FlowMuse 需求文档

## 1. 产品定位

FlowMuse 是一款**跨平台协同白板（笔记）应用**，面向需要手写、绘图、多人协作的场景（教学、会议、设计草稿、个人笔记）。核心差异化：自研白板内核 + 多端原生体验 + 手写笔压感 + 实时协作。

## 2. 目标用户与场景

- **学生/研究者**：课堂笔记、公式推导、思维导图
- **团队**：会议白板、方案草稿、实时协同标注
- **个人**：手写日记、灵感速记

典型场景：一台平板用手写笔书写压感笔迹，其他成员通过手机/电脑实时查看与编辑同一白板。

## 4. 功能需求

### 4.1 白板编辑（核心）

| 需求 | 说明 |
|---|---|
| 自由画笔 | 支持压感变粗、平滑曲线；鼠标/触摸无压感时用速度模拟；首次选择时打开笔型面板，后续可恢复上次笔型及完整参数 |
| 形状工具 | 矩形、椭圆、菱形、直线、箭头，支持手绘风（rough）|
| 文本 | 文本框输入与编辑 |
| 橡皮 | 笔划擦除 |
| 选择/移动/缩放 | 单选、多选、框选、拖动、缩放、旋转 |
| 撤销/重做 | 历史栈 |
| 图层与分组 | 元素分组、画框、图层管理 |
| 对齐 | 多种对齐与分布 |
| 激光笔 | 演示用临时笔迹 |

### 4.2 笔记库管理

| 需求 | 说明 |
|---|---|
| 笔记 CRUD | 创建、打开、重命名、删除 |
| 文件夹与标签 | 分类组织 |
| 列表视图 | 全部/笔记/PDF 等筛选 |
| 封面 | 缩略图/色块 |
| 搜索 | 按标题检索 |

### 4.3 协作（实时多端）

| 需求 | 说明 |
|---|---|
| 房间 | 创建/加入协作房间 |
| 实时同步 | 笔迹与元素的增量同步（Socket.IO）|
| 协作者感知 | 在线协作者列表/光标 |
| 场景存储 | 服务端持久化白板场景 |
| 端到端加密 | 场景内容加密存储 |

### 4.4 账号

| 需求 | 说明 |
|---|---|
| 登录/会话 | 账号认证、token 安全存储 |
| 身份 | 协作中的用户身份标识 |

### 4.5 AI 手写识别

| 需求 | 说明 |
|---|---|
| 自动识别 | 手写完成后自动调度识别（1秒去抖延迟）|
| 手动转换 | 框选笔迹后手动触发转换为文本/图形 |
| 多笔形识别 | 支持 text / shape / formula / drawing 等 hint 提示 |
| 服务端识别 | 通过 HTTP POST 向识别服务发送笔迹数据，返回结构化元素 |

### 4.6 PDF 导入

| 需求 | 说明 |
|---|---|
| 新建笔记导入 | 创建笔记时选择 PDF 文件作为画布背景 |
| PDF 页面渲染 | 逐页渲染 PDF 为图片嵌入白板，支持分页模式 |
| 鸿蒙原生渲染 | OHOS 端通过 PDFKit 原生渲染页面，非平台通用方案 |

### 4.7 分页模式

| 需求 | 说明 |
|---|---|
| 分页视口 | 页面式纵向滚动布局，模拟纸质笔记本体验 |
| 添加页面 | 底部弹性拖拽添页，带释放阈值和弹簧动画 |
| 页面跳转 | 滚动到指定页、追加空白页 |
| 进度指示 | 右侧滚动条显示当前页面进度 |
| 背景页 | 支持 PDF 页面 / 空白模板页作为画布背景 |

### 4.8 笔形管理

| 需求 | 说明 |
|---|---|
| 独立笔状态 | 每种笔形（钢笔、铅笔、荧光笔等）独立保存颜色/粗细/压感 |
| 切换恢复 | 切换笔形时自动恢复上次使用的参数 |
| 铅笔着色器 | 铅笔使用 Fragment Shader 模拟纸质纹理效果 |

### 4.9 编辑器偏好设置

| 需求 | 说明 |
|---|---|
| 文档偏好 | 保存自动保存间隔、默认文档类型、页面模板和页面排列方向 |
| 工具偏好 | 保存默认工具、默认笔形以及每种笔形的颜色、粗细和压感灵敏度 |
| 手写笔偏好 | 支持关闭真实压感、选择三档压感曲线及启停防误触 |
| 手势偏好 | 支持启停双指缩放和非抓手模式下的单指平移 |
| 跨端持久化 | 使用本地 `local_settings` 保存，不改变白板场景和协作协议 |

### 4.10 语音转文字

| 需求 | 说明 |
|---|---|
| 跨端语音输入 | Android 使用系统 SpeechRecognizer、鸿蒙使用 Core Speech Kit、Web 使用 Web Speech API |
| 实时预览 | 中间识别结果仅在本地浮层展示，不修改白板场景 |
| 标准文本落地 | 最终结果一次性创建普通 TextElement，复用撤销、保存、导出和协作 |
| 生命周期安全 | 取消、页面退出或应用退后台后立即释放麦克风，不提交临时文字 |
| 隐私 | 默认不保存、上传或同步录音；首次使用明确提示可能由平台识别服务处理 |
| 安全降级 | 权限拒绝、平台不支持和识别错误不影响白板继续使用 |

### 4.11 鸿蒙端特有

| 需求 | 说明 |
|---|---|
| 手写笔压感 | 华为手写笔的 force/pressure 驱动笔迹粗细 |
| 鸿蒙编译 | 通过 vendor code_assets 解决 ohos native-assets 兼容 |
| 原生 HTTP | Platform Channel → @ohos.net.http，绕过 dart:io socket 缺陷 |
| 原生文件选择 | Platform Channel → DocumentViewPicker，替代未注册的 file_picker 插件 |
| 原生文件保存 | Platform Channel → DocumentViewPicker(DOWNLOAD)，直存 Downloads 目录 |
| 原生 PDF 渲染 | Platform Channel → PDFKit，替代 pdfx 插件渲染 PDF 页面 |
| 原生语音识别 | Platform Channel → Core Speech Kit，将普通话转换为标准白板文本元素 |
| 网络安全 | network_config.json 显式允许 cleartext HTTP 流量 |
| 桌面服务卡片 | 鸿蒙桌面显示最近白板并一键继续创作；无最近白板时回到资料库 |

## 5. 非功能需求

| 维度 | 要求 |
|---|---|
| 跨端一致性 | 一份 Dart 代码，各端行为一致；平台特有能力（压感）优雅降级 |
| 性能 | 笔迹低延迟跟手；大场景下滚动/缩放流畅 |
| 离线 | 单机可用（无服务端也能本地书写与存储）|
| 数据格式 | Excalidraw JSON 兼容（可导入/导出 Excalidraw）|
| 可部署 | 服务端 Docker 一站式部署 |

## 6. 当前状态与演进

项目按里程碑演进，关键节点：

1. **组件化框架** — Riverpod + go_router + feature 分层（已完成）
2. **markdraw 内核** — 自研 Excalidraw 风格白板（已完成）
3. **协作后端** — Go + Socket.IO + 房间/加密（已完成）
4. **鸿蒙适配** — ohos 端编译、原生 Platform Channel 桥接（HTTP/文件选择/PDF渲染）（已完成）
5. **手写笔压感** — PointerEvent.pressure + 多笔形独立状态管理（已完成）
6. **AI 手写识别** — 服务端识别流水线，自动/手动双模式（已完成）
7. **分页模式** — 分页视口、PDF 背景页、弹性添页交互（已完成）
8. **持续迭代** — UI 打磨、性能优化、协作稳定性

详细设计、计划与验收记录见 `docs/`。
