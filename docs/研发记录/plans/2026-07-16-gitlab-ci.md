# GitLab 质量门禁实施计划

**目标：** 用 `.gitlab-ci.yml` 在 GitLab 上执行 Flutter 与 Go 质量门禁。

## 实施

- [x] 新建根目录 `.gitlab-ci.yml`。
- [x] 使用 `ohos` Runner 执行 Flutter 分析与测试。
- [x] 使用 `linux` Runner 执行 Go 测试与 vet。
- [x] 移除未配置或不兼容的六端构建 job，避免阻塞质量门禁。
- [ ] 以真实 Runner 验证两项质量 job 全绿。

平台构建待具备对应 Runner、SDK 与标准 Flutter/鸿蒙 Flutter 的编译兼容性后再单独恢复。
