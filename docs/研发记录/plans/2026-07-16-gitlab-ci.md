# GitLab 六端持续集成实施计划

**目标：** 用 `.gitlab-ci.yml` 在 GitLab 上执行质量门禁和六端构建。

## 实施

- [x] 新建根目录 `.gitlab-ci.yml`。
- [x] 使用 `linux` Runner 执行 Flutter 分析/测试、Go 测试/vet、Android 和 Web 构建。
- [x] 使用 `macos` Runner 构建无签名 iOS app 和 macOS app。
- [x] 使用 `windows` Runner 构建 Windows Release。
- [x] 使用 `ohos` Runner 构建鸿蒙 HAP，并归档 `build/ohos/hap/entry-default-signed.hap`。
- [x] 为所有 build job 设置七天 artifact 保留期。
- [ ] 在 GitLab Pipeline Editor 运行 CI Lint，并以真实 Runner 验证流水线。
