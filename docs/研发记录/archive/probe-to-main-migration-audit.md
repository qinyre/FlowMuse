# probe 到 main 迁移审计

审计日期：2026-07-09

迁移原则：

- 按 `markdraw-harmonyos-probe` 的第一父提交时间顺序审查。
- main 已有同等或更新实现时不重复迁移。
- 与 main 当前架构冲突的实现按功能重写，不直接 cherry-pick。
- PDF 导入沿用“新建文件 → 导入”入口，不增加白板菜单或资料库独立入口。

## 验证边界

- 迁移专项测试覆盖压感、手写笔平滑、PDF 导入、PDF 背景选择、画布约束、SVG 导出和 OHOS 文件通道。
- HAP 已完成 Flutter、ArkTS 编译和打包检查；签名阶段因本机 keystore 密码不匹配失败。
- OHOS 文件选择、文件保存和 SQLite 持久化仍需要真机回归。
- 仓库既有资料库组件测试依赖 `.env` 初始化，目前不作为本次迁移功能的通过依据。
