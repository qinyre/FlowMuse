# 架构约束

## Excalidraw 对齐原则

FlowMuse 的绘画协作核心继续采用 Excalidraw 的方案。除手绘视觉风格外，后续实现应尽量与 Excalidraw 完全对齐。

- 数据模型、场景 JSON、元素字段、版本字段、顺序字段、删除保留策略优先与 Excalidraw 保持兼容。
- 画布交互语义、选择、编辑、拖拽、缩放、文本、图片、Frame、绑定箭头、分组等白板能力以 Excalidraw 为目标实现。
- 实时协作、端到端加密、房间链接、冲突合并、远端持久化、光标与在线状态同步优先参考 Excalidraw 的架构和算法。
- 导入导出优先兼容 Excalidraw 文件格式和场景恢复语义。
- 手绘风格不是强制目标，可以采用更符合 FlowMuse 产品定位的非手绘视觉渲染，但不得破坏 Excalidraw 数据与协作兼容性。
