# 白板手指绘制设计

## 目标

在白板页面提供持久化的“手指绘制”开关，使触摸屏用户可在不影响手写笔输入的情况下选择用手指绘图。

## 交互

- 开关位于白板顶部导航栏“已保存”状态右侧，默认关闭，关闭时保持现有单指平移行为。
- 开启后，单指触摸交给当前绘图工具；选择自由绘制工具时生成手写笔迹。
- 开启后，双指手势首次被判定为捏合缩放或双指平移后保持该模式直到结束；它会取消正在进行的单指笔迹，避免笔迹与视口同时变化。
- 关闭时，既有“双指缩放”偏好仍按原行为处理。

## 状态与边界

- 复用 `EditorPreferences`、`EditorPreferencesViewModel` 及 `local_settings` 的既有持久化链路，不新增存储或依赖。
- 开关仅影响本地输入分发和视口，不改变场景 JSON、协作消息或平台适配层，因此保持 Excalidraw 与跨端兼容。
- 分页画布的单指触摸拦截必须在开关开启时让出给编辑器，双指仍由统一的 GestureDetector 处理。

## 验证

- 偏好序列化往返和缺失字段默认值正确。
- 开启时触摸自由绘制能创建笔迹；双指缩放与平移在单次手势中互斥。
- 顶部导航栏在紧凑与桌面布局均显示可操作的开关。
