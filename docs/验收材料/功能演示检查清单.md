## 💡 答辩预期问题与回答

### 1. 技术类问题

**Q1：FlowMuse 与 Excalidraw 的核心区别是什么？**
**A1**：
三大核心区别：
1. **端到端加密**：FlowMuse 采用 AES-GCM-128 客户端加密，服务端零知识；Excalidraw 云端明文存储
2. **鸿蒙深度适配**：FlowMuse 集成 7 个鸿蒙原生特性（Pen Kit、Core Speech Kit 等）；Excalidraw 不支持鸿蒙
3. **跨平台统一**：FlowMuse 一份代码 6 平台 95% 复用；Excalidraw 主要是 Web 端

**Q2：端到端加密协作的技术难点是什么？**
**A2**：
核心难点是**安全与性能的平衡**：
1. **冲突解决**：多人同时编辑加密内容，服务端无法解密比对，需要客户端乐观锁 + 版本号仲裁
2. **增量同步**：不能每次传输完整场景（太大），需要增量 diff，但加密后无法服务端 diff
3. **密钥管理**：房间密钥如何安全分发给新加入者，同时防止中间人攻击

我们的解决方案：
- 使用 baseSceneVersion + SHA256 哈希做乐观锁
- 客户端生成增量 patch，加密后传输
- 房间密钥通过 QR Code 或房间码离线分发

**Q3：如何保证 6 平台的一致性？**
**A3**：
三层策略：
1. **架构隔离**：平台差异收敛到适配层，共享代码禁止 `Platform.is*` 判断
2. **测试覆盖**：224 个单元测试，Repository 层 100% 覆盖，核心逻辑跨平台一致
3. **真机验证**：4 设备（鸿蒙平板/手机 + Android + PC）协作压测

**Q4：鸿蒙 Platform Channel 适配的最大挑战是什么？**
**A4**：
最大挑战是**鸿蒙 Flutter 是社区移植版**，部分 API 与标准 Flutter 不一致：
1. **API 缺失**：FragmentProgram/Shader 不支持，需要降级方案
2. **路径约定不同**：`ohos/` 替代 `android/`/`ios/`
3. **插件生态薄弱**：file_picker、shared_preferences 需要 fork 或自己实现

我们的应对：
- 建立 `harmonyos-guides/` 知识库记录适配经验
- 使用 Platform Channel 直接调用鸿蒙原生 API
- 优雅降级：鸿蒙特性失败不影响核心功能

### 2. 产品类问题

**Q5：目标用户是谁？如何获客？**
**A5**：
三大用户群：
1. **高校师生**（主要市场）：1000 万+ 潜在用户
   - 获客策略：10 所重点高校合作试点，校园大使计划
2. **企业团队**（增长市场）：500 万+ 潜在用户
   - 获客策略：私有部署 POC，与 CRM/OA 集成
3. **个人创作者**（细分市场）：300 万+ 潜在用户
   - 获客策略：开源社区、设计师社群

初期聚焦高校，通过学生口碑传播，建立品牌影响力。

**Q6：商业模式是什么？如何盈利？**
**A6**：
三层变现模式：
1. **免费版**：本地存储无限 + 3 人协作 → 获客 100 万用户
2. **个人高级版**（¥9.9/月）：云端同步 + 无限协作 + 高级 AI → 5% 付费率，495 万元/年
3. **企业版**（¥199/用户/年）：私有部署 + SSO + 审计 → 500 家企业，498 万元/年

**总收入预期**：1000 万元/年（3 年目标）

**Q7：如何应对 Miro、Notion 等巨头竞争？**
**A7**：
差异化策略：
1. **隐私优先**：唯一的端到端加密协作白板，瞄准隐私敏感用户（企业、研究者）
2. **鸿蒙首选**：深度适配鸿蒙生态，Miro/Notion 短期内无法跟进
3. **开源策略**：开源核心组件，建立开发者社区护城河
4. **本地化**：中国市场优先，Miro 国际版网络不稳定

避免正面竞争，切细分市场（隐私 + 鸿蒙 + 教育），建立壁垒后再扩展。

1. **短期**（6 个月内）：开源 Platform Channel 参考实现
